# EA-Pathways
EA-Pathways identifies genes and pathways with EA distributions that deviate from the expected Boltzmann distribution of the background population of variants. It can be applied to somatic or ultra-rare germline variants (e.g., missense, fs-indel, nonsense). Variants in VCF format should be annotated with Ensembl's VEP software and must have the following INFO fields: Consequence, SYMBOL, ENSP, HGVSp, Ensembl_proteinid, AC (allele count). Variants must also have an INFO field with EA score annotations.<br>

## Requirements
### Hardware Requirements
EA-Pathways can be ran on a standard laptop or desktop with 16GB+ RAM. However, with larger cohorts comprised of more than a few thousand samples (2k+ samples) users should run on a local or cloud computing cluster with 20+ cores.<br>

### Software Dependencies
Required packages are included in ea-pathways.yml. <br>

## Installation
The repository can be cloned using: git@github.com:LichtargeLab/EAML.git


## Usage
To run EA-Pathways from a cohort VCF file, use the following command: <br>
<pre><code> python EA_Pathways.py --AnalysisName ExpName --BiologicalGroups test_cohort_input/Reactome2023_Greater5Less100_03032023.csv --Output outputDirectory --Cores 10 --VCF cohortVCF --Samples samples.txt --RefPopAC reference_populaton_INFO.txt </code></pre>

### Required command line parameters for running from VCF:<br>
--AnalysisName = prefix added to all output files <br>
--VCF = location of cohort VCF with VEP and EA annotations<br>
--BiologicalGroups = location of file containing biological groups of interest (e.g., Reactomes, GO terms)<br>
--Output = output directory<br>
--Samples = single column txt file with sample IDs from cohort VCF<br>

### Optional command line parameters:<br>
--Cores = number of cores for parallelization of analysis; default = 1<br>
--MinAC = minimun allele count considered in analysis; default = 1<br>
--MaxAC = maximum allele count considered in analysis; default = 5<br>
--CollectGT = set to 1 to collect genotypes of variants considered in analysis; default = None<br>
--Verbose = set to Y to collect intermediary log files generated by analysis; default = N<br>
--Variants = location of cohort variants that have been pre-parsed from VCF; default = N<br>
--RefPopAC = location of txt file with variant allele counts based on reference population; is not included, analysis will proceed with cohort singletons <br>

## Additional Notes
<b>For faster analyses --</b><br>
EA-Pathways sorts cohort variants into genes and pathways. This is not a sample-level analysis. If the user does not need to associate sample IDs with cohort variants (i.e., does not need to know sample genotypes) then it is highly recommended to run the pipeline without calling the "Collect_GT" flag. When calling "Collect_GT," the majority of the analysis run time is dedicated to parsing genotypes from the input VCF. This may take several hours depending on degree of parallelization.

<b>Reference population variant file format --</b><br>
If reference population variants are in VCF format, then user can use the following command to generate the reference population variant file:<br>
<pre><code>bcftools query -f '%CHROM\t%POS\t%REF\t%ALT\t%INFO/AC\n' reference_population.vcf.gz > reference_populaton_INFO.txt </code></pre>
Reference file txt should be tab separated with the following columns and no header:<br>
Column 1, "CHROM": variant chromosome location <br>
Column 2, "POS": variant start location <br>
Column 3, "REF": variant reference allele <br>
Column 4, "ALT": variant alternative allele <br>
Column 5, "AC": variant allele count in reference population <br>
